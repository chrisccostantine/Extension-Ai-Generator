# Embedded App Starter

This `web` folder is the target embedded-app surface.

In the real Shopify app generated by Shopify CLI, this area will be replaced by the framework scaffold that ships with the template. Keep these files as a reference for the data flow we want:

- identify the installed shop
- request current plan and usage from the backend
- submit manual plan requests
- trigger AI generation for the current product

## Current contract with backend

- `GET /shop-status?clientId=shopify-store:<shop-handle>`
- `GET /plans`
- `POST /plan-requests`
- `POST /generate-product-content`

## Target improvement

After full Shopify auth is implemented, remove `BACKEND_EXTENSION_TOKEN` from the flow and replace it with trusted Shopify session/shop validation on the backend.
